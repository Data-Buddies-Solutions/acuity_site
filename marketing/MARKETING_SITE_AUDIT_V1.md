# Acuity Health Site Audit V1

## 1. Purpose

This audit reviews the current public site against the Acuity marketing strategy:

- category: `patient access and engagement for eye care`
- wedge: `patient access / AI-native front desk`
- expansion: `engagement, texting, follow-up, communication layer`
- tone: calm, credible, operational, specialty-aware

The objective is to identify what is working, what is still weak, and what the highest-leverage next moves are.

## 2. Current Site Assessment

### Overall Direction

The site is now directionally strong.

The homepage, platform page, results page, and insights section are materially more coherent than before.
The site now presents Acuity as a focused eye-care communications platform rather than a generic AI tool vendor.

### What Is Working

#### 1. The category framing is much stronger

The homepage now leads with:

- patient access
- patient engagement
- front-desk overload
- eye-care specificity

This is strategically correct.
It aligns with the market pain more directly than `AI phone receptionist`.

#### 2. The offer architecture is coherent

The `/platform` page gives the market a cleaner mental model:

- `Acuity Access`
- `Acuity Engage`
- `Enterprise / Multi-Location`

This is a better buying story than a flat menu of unrelated tools.

#### 3. The proof path exists

The `/results` page creates a dedicated place for:

- outcomes
- proof points
- operator metrics
- customer quotes
- impact framing

This is a meaningful upgrade over relying on homepage claims alone.

#### 4. The off-strategy blog drift is contained

The rename to `Insights` and the removal of the old general AI topics was the right move.
This preserves category discipline.

#### 5. Tone is more professional and market-aware

The tone is now much closer to:

- operational
- serious
- healthcare-adjacent
- specialty-aware

That is the correct voice for ophthalmology and optometry buyers.

## 3. Gaps That Still Matter

### 1. Proof is still lighter than it should be

The site now has a results page, but much of the proof is still:

- illustrative
- benchmark-style
- claim-oriented

What is missing:

- hard case studies
- named or semi-anonymized before/after metrics
- screenshots of real workflows
- a stronger trust stack beyond general statements

This is now the biggest conversion gap.

### 2. The site still needs segment-specific pages

Right now the site is mostly:

- homepage
- platform
- results
- FAQ
- insights

That is a good foundation, but it still lacks high-intent entry pages for specific buyer and problem searches.

Most important missing pages:

- `Patient Access`
- `Patient Engagement`
- `Ophthalmology`
- `Optometry`
- `Compare vs voicemail / answering service / call center`

These pages would strengthen both SEO and sales relevance.

### 3. The homepage still leans a little abstract at the top

The homepage is much better, but the hero still benefits from stronger business-outcome reinforcement immediately below the headline.

Specifically:

- the main thesis is strong
- the trust signals are present
- but the proof density above the fold can still increase

Adding one short proof strip under the hero copy would help:

- calls answered
- hours returned
- languages supported
- specialty focus

### 4. The results page should evolve from framed proof to harder proof

The current `/results` page is good for stage one.
But stage two should move it toward:

- case-study blocks
- deployment metrics
- benchmark methodology
- what changed in 30 / 60 / 90 days

Right now it is a strong narrative page.
It should become a stronger sales-conversion page.

### 5. FAQ should be more aligned with the package architecture

The FAQ still answers useful operational questions, but it should eventually reflect:

- `Acuity Access`
- `Acuity Engage`
- platform fee logic
- implementation expectations
- what is included vs add-on vs custom work

It currently feels more old-product than new-platform in structure.

### 6. Insights is intentionally empty, but needs a relaunch plan

This is fine for now.
But once the first 4 operator-relevant articles exist, the page should relaunch with:

- a stronger editorial intro
- 4 high-value insights
- one featured benchmark or playbook

Without that, the section remains a placeholder rather than an authority asset.

## 4. Highest-Leverage Next Steps

## Priority 1: Build one real proof asset

Best next move:

- one case study page
- one ROI one-pager
- one visual workflow proof artifact

If only one gets done first:

`Build a real case-study/results asset`

Reason:
Proof is now the main bottleneck between interest and trust.

## Priority 2: Add two high-intent solution pages

Best next pages:

- `Patient Access`
- `Patient Engagement`

Reason:

- they reinforce the category
- they give the sales team more direct landing pages
- they let the site rank and convert around the right problems

## Priority 3: Tighten homepage proof density

Add:

- one proof strip under hero
- one specialty-specific trust block
- one stronger link path into results or case study content

Reason:

- homepage is the brand anchor
- more proof above the fold will improve conversion quality

## Priority 4: Rework FAQ around the new product architecture

The FAQ should eventually answer:

- what is Acuity Access
- what is Acuity Engage
- how pricing works
- what implementation includes
- what counts as an add-on
- how multi-location works

Reason:

- buyers use FAQ for risk reduction
- this is where product architecture needs to become operationally clear

## Priority 5: Relaunch Insights with the first 4 articles

Recommended first 4 insights:

- The Hidden Cost of Missed Calls in Ophthalmology Practices
- How Eye Care Practices Should Measure Patient Access
- Front-Desk Overload: What to Fix First
- Patient Engagement Starts Before the Visit

Reason:

- this turns Insights from placeholder to authority asset
- it supports LinkedIn distribution
- it keeps the site's content aligned with the category

## 5. Suggested 30-Day Site Roadmap

### Week 1

- create one case study / proof asset
- define proof metrics and screenshots to collect

### Week 2

- build `Patient Access` page
- build `Patient Engagement` page

### Week 3

- revise FAQ around Access / Engage / implementation / pricing logic
- add stronger proof strip to homepage

### Week 4

- publish first 2-4 insights
- tighten results page with harder evidence where available

## 6. Strategic Recommendation

The site is now good enough to support outbound, founder-led LinkedIn, and early inbound interest.

The next phase should not be another broad messaging rewrite.
The next phase should be:

- `proof`
- `solution pages`
- `authority content`

That is what will make the site more credible, more searchable, and more sales-useful.

## 7. Final Assessment

### Current State

`Strong strategic direction, moderate proof depth, good offer clarity, incomplete solution coverage`

### Best Next Move

`Build proof first, then build the two solution pages, then relaunch insights with targeted content.`
