# Issue tracker

Issues and PRDs live in GitHub Issues for
`Data-Buddies-Solutions/acuity_site`. Use `gh` from this repository.

- Read: `gh issue view <number> --comments`
- List: `gh issue list`
- Create: `gh issue create`
- Comment: `gh issue comment <number>`
- Label: `gh issue edit <number> --add-label "<label>"`
- Close: `gh issue close <number> --comment "<summary>"`

Pull requests are not a triage request surface.

When a skill says “publish to the issue tracker,” create a GitHub issue.
When it says “fetch the relevant ticket,” read the issue and its comments.
