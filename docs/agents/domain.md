# Domain docs

This is a single-context repository.

Before changing domain behavior:

- Read root `CONTEXT.md` when it exists.
- Read relevant decisions under `docs/adr/` when they exist.
- Use the domain vocabulary defined in `CONTEXT.md`.
- Explicitly identify proposed work that conflicts with an ADR.

Missing domain files are not an error. Create them lazily through the
domain-modeling workflow when terminology or a durable decision is resolved.
